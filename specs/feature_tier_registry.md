# Feature Tier Registry
**Status:** ACTIVE — updated as features are defined

This is the canonical record of which platform features are free vs. paid.
Lens-level tiers are tracked separately in memory: `project_lens_tiers.md`.

---

## Tier Definitions

| Tier | Description |
|------|-------------|
| **Free** | Available to all users, no gate |
| **Paid** | Requires active subscription — gated at point of access |
| **Paid — Lens-gated** | Available only to users with a premium lens subscription |

---

## Alerts / Notification Layer

| Feature | Tier | Notes |
|---------|------|-------|
| In-platform indicator (dot, count) | Free | Basic presence only |
| Single convergence state alert per active search | Free | Fires on what IS — not what's coming |
| Foresight window alerts (T-minus X hours) | **Paid** | The "Know First" edge. Core premium value proposition. |
| Lens-specific threshold rules | **Paid — Lens-gated** | Retirement vs. Investor fire differently. Tied to lens access. |
| Multi-signal watch lists | **Paid** | Monitoring more than one profile simultaneously |
| Email / push delivery | **Paid** | Off-platform reach |
| Custom delivery cadence | **Paid** | Frequency control |
